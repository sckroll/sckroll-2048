# v2 전환 계획

## 전환 원칙

- 현행 동작을 특성 테스트로 고정한 뒤 구조 교체
- 규칙, 이력, 렌더링, 디자인의 순차 도입
- 각 단계의 독립 실행과 되돌릴 수 있는 커밋
- 레거시 <code>js</code> 구현을 비교 기준으로 유지
- 새 <code>src</code> 진입점 검증 완료 뒤 레거시 제거
- 문서와 코드의 같은 커밋 갱신

## 시작 전 제품 결정

| 항목 | 권장안 | 결정 필요 이유 |
|---|---|---|
| 2·4 생성 확률 | 90%·10% | v1의 50%·50%와 플레이 난도 차이 |
| 사용자 목표 상한 | 2³⁰ | 점수 정밀도와 UI 표시 범위 |
| 이력 상한 | 10,000개 | 높은 목표와 메모리 균형 |
| 최고 점수 | 목표별 분리 | 난도와 세션 길이 차이 |
| 목표·패배 동시 발생 | 목표 달성 우선 | 중복 대화상자 제거 |
| v1 데이터 승계 | 테마와 최고 점수 | 사용자 설정 보존 |

## 단계 0: 기준선 고정

### 작업

- 기존 버전의 핵심 브라우저 흐름과 기대 결과 기록
- 기준 이동 함수와 현재 구현의 대조 데이터셋 기록
- 승리·패배 동시 재현과 짧은 드래그 오입력 절차 기록
- 현재 화면 크기의 비교 이미지 확보
- 성능 측정 데이터셋과 실행 방법 기록

### 종료 조건

- 네 방향 대표 규칙과 무작위 보드 대조 결과 보존
- 알려진 결함의 재현 입력·화면 크기·기대 결과 명시
- 도구 기반 추가 뒤 자동화할 테스트 목록 확정

## 단계 1: 도구 기반

### 작업

- <code>package.json</code>, 잠금 파일, <code>.gitignore</code> 추가
- TypeScript 최신 안정 버전 재확인과 정확 버전 고정
- Vite 계열 빌드, 테스트, 린트, 포맷 구성
- 엄격한 <code>tsconfig</code> 적용
- 0단계 기준선의 특성 테스트 자동화
- 개발·빌드·미리보기 명령 문서화
- CI의 타입 검사, 단위 테스트, 프로덕션 빌드 추가

### 종료 조건

- 빈 TypeScript 진입점의 개발 서버와 프로덕션 빌드 성공
- 레거시 특성 테스트와 기준 보드 대조 성공
- 타입 오류 0건
- 잠금 파일 기반의 재현 설치 성공

## 단계 2: 순수 게임 엔진

### 작업

- <code>Direction</code>, <code>Tile</code>, <code>GameState</code>, <code>TurnResult</code> 정의
- 압축·병합·점수 계산을 순수 함수로 구현
- 주입 가능한 난수와 생성 정책 구현
- 목표 달성·계속·게임 오버 단계 구현
- 빈 셀과 최대 타일의 파생 계산 구현

### 종료 조건

- DOM 없는 단위 테스트 실행
- 기준 구현과 무작위 보드 대조 성공
- 이동·병합 합 보존, 생성 뒤 합 증가, 점수, 한 턴 한 번 병합의 속성 테스트 성공
- 이동 불가 입력에서 난수 호출 0회 확인

## 단계 3: 이력과 컨트롤러

### 작업

- 스냅샷·항목·커서 기반 <code>HistoryStore</code> 구현
- <code>GameController</code>의 명령 처리와 상태 커밋 구현
- undo, redo, 미래 분기 제거, 오래된 첫 항목·스냅샷 제거와 커서 보정 구현
- 최고 점수의 이력 외부 단조 증가 처리
- 대화상자 상태를 게임 단계에서 도출

### 종료 조건

- 임의 시퀀스의 undo·redo 왕복 상태 일치
- redo의 난수 호출 0회
- undo 뒤 새 이동의 미래 상태 제거
- 승리·패배 상태의 왕복 복원
- 10,000개 이력의 커서 동작과 제거 정책 확인

## 단계 4: Canvas 렌더러

### 작업

- DPR 대응 크기 조정
- 보드, 빈 셀, 타일, 숫자 그리기
- 안정적인 타일 ID 기반 이동 전환
- 병합·생성 애니메이션
- 모션 감소와 탭 비활성화 복구
- 렌더러의 <code>mount</code>·<code>destroy</code> 구현

### 종료 조건

- DPR 1·2·3의 선명도 확인
- 크기 변경과 회전 뒤 좌표 정합성 확인
- 프레임 루프 중복과 폐기 후 콜백 부재
- 최종 Canvas 상태와 엔진 상태 일치

## 단계 5: 입력과 접근 가능한 화면

### 작업

- Pointer Events와 키보드 입력 제어기 구현
- HUD, 도구 모음, 목표 선택, 대화상자 DOM 구현
- 숨김 보드 표현과 live region 구현
- 포커스 이동·트랩·복귀 구현
- undo/redo 단축키와 비활성 상태 구현

### 종료 조건

- 키보드 전용 새 게임부터 승패까지 완료
- 보드 밖 포인터 입력과 짧은 드래그의 무시
- 포인터 취소 뒤 좌표 잔존 부재
- 대화상자 한 개와 올바른 포커스 유지
- 모션 감소 환경의 즉시 전환

## 단계 6: 이력 화면과 새 디자인

### 작업

- 데스크톱 이력 패널과 모바일 하단 시트 구현
- 현재·과거·미래 항목 시각 구분
- 가상 목록과 이벤트 위임 적용
- 반응형 게임 셸과 디자인 토큰 구현
- HUD 점수 증가 애니메이션 구현
- 동적 타일 팔레트와 숫자 크기 구현
- 시스템·라이트·다크 테마 구현

### 종료 조건

- 열린 이력 패널과 보드 겹침 부재
- 10,000개 이력에서 제한된 DOM 노드 수

- 320px 세로 화면과 844×390 가로 화면 사용 가능
