# v2 기능 명세

## 제품 목표

현재 게임의 단순한 조작성을 유지하면서 목표 선택, 정확한 이력 탐색, Canvas 기반 모션, 현대적인 반응형 화면을 제공하는 버전임.
규칙과 화면의 분리를 통해 기능 추가와 회귀 검증 비용을 낮추는 목표임.

## 기능 범위

| 기능 | v2 필수 범위 |
|---|---|
| TypeScript | 7.0.2 기준의 엄격 모드와 정확한 버전 고정 |
| Canvas | 보드, 타일, 숫자, 이동·병합·생성 모션 |
| 새 디자인 | 반응형 게임 셸, 목표 표시, 이력 패널·시트 |
| undo/redo | 성공 이동 단위의 이전·다음 상태 복원 |
| 목표 타일 | 2048, 4096, 8192, 16384 이상 |
| 접근성 | 키보드, 포커스, 상태 알림, 모션 감소 |
| 저장 | 테마, 기본 목표, 목표별 최고 점수 |

## 새 게임과 목표 설정

- 시작 화면과 설정 화면에 목표 선택 제공
- 기본 프리셋 2048, 4096, 8192, 16384 제공
- 사용자 지정 목표는 2의 거듭제곱만 허용
- 권장 허용 범위는 2¹¹부터 2³⁰까지
- 숫자 직접 입력보다 지수 단계 증감 컨트롤 우선
- 현재 게임 중 목표 변경 시 새 게임 확인 대화상자 표시
- 확인 전 기존 보드와 이력 유지
- 목표 변경 확정 시 새 상태와 빈 이력 생성
- 목표별 최고 점수 분리
- 현재 최대 타일과 목표를 HUD에 함께 표시

2³⁰을 넘는 목표는 점수 정밀도와 표시 정책 검토 뒤 확장할 후속 범위임.

## 타일 생성 규칙

현행은 2와 4가 각각 50%인 규칙임.
v2 권장 기본값은 관례적인 2의 90%, 4의 10% 정책임.
최종 확률은 구현 전에 제품 결정으로 고정하고 테스트에 명시해야 함.

- 빈 셀 선택의 균등 분포
- 유효 이동 뒤 한 번의 생성
- 이동 불가 입력에서 생성 금지
- 테스트용 난수 공급원 주입
- redo에서 저장된 생성 결과 사용
- 빈 셀 없는 생성 요청의 명시적 오류 또는 안전한 무변경 반환

## 이동과 점수

- 방향당 압축 한 번과 병합 한 번
- 같은 타일의 한 턴 두 번 병합 금지
- 병합 결과 타일 값만큼 점수 증가
- 이동만 발생한 턴의 점수 증가 0
- 이동 불가 입력의 점수·턴·이력 불변
- 성공 이동 완료 상태의 턴 증가
- 최고 점수의 즉시 단조 증가
- undo 시 현재 점수 복원과 최고 점수 유지

## 승리와 계속하기

- 최대 타일이 선택 목표 이상을 처음 만족할 때 <code>won</code> 단계
- 목표 달성 대화상자 한 개 표시
- 새 게임, undo, 계속하기 선택 제공
- 추가 이동 가능한 보드에서만 계속하기 활성화
- 계속 선택 시 새 항목·턴 증가 없이 현재 커서 스냅샷을 <code>continued</code> 단계로 교체
- 계속한 세션에서 같은 목표 대화상자의 재표시 금지
- 계속 선택 전 undo로 목표 달성 전 상태 복원 가능
- 계속 선택 전 redo는 달성 대화상자, 선택 후 이력 탐색은 <code>continued</code> 상태 복원
- 목표 달성과 이동 불가가 겹친 경우 달성 결과 우선 표시

## 게임 오버

- 새 타일 생성 뒤 빈 셀과 인접 병합 가능성 검사
- 목표 신규 달성 상태가 없는 이동 불가 보드에서 <code>lost</code> 단계
- 게임 오버 대화상자 한 개 표시
- 새 게임과 undo 제공
- undo 뒤 대화상자 닫힘과 플레이 상태 복원
- redo 뒤 게임 오버 상태 복원

## 이력과 로그

### 기록 단위

성공 이동 한 번을 이력 항목 하나로 기록함.
각 항목에 방향, 점수 증가, 생성 타일, 타일 전환 정보를 보관함.
각 경계에 완전한 <code>GameState</code> 스냅샷을 보관함.

### undo

- 버튼과 <code>Ctrl/Cmd+Z</code> 제공
- 커서가 0일 때 비활성화
- 애니메이션 큐 정리 후 이전 스냅샷 복원
- 승리·패배 대화상자 상태도 스냅샷에서 복원
- 짧은 역방향 전환 또는 모션 감소 시 즉시 표시

### redo

- 버튼과 <code>Ctrl/Cmd+Shift+Z</code>, <code>Ctrl+Y</code> 제공
- 마지막 스냅샷에서 비활성화
- 난수 호출 없이 다음 스냅샷 복원
- 원래 생성 타일과 점수·턴 완전 복원

### 분기

- undo 뒤 새 방향 입력 시 미래 이력 제거
- 제거 전 별도 확인은 생략하는 일반 편집기 방식
- redo 버튼 즉시 비활성화
- 로그에서 제거된 미래 항목 삭제

### 규모

- 보존 이력 최대 10,000개 성공 이동 권장
- 새 이동에서 미래 분기 제거, 항목 추가, 보존 상한 적용 순서
- 상한 초과 시 첫 항목과 첫 스냅샷 동시 제거, 커서 1 감소
- 제거 발생을 이력 패널에 한 번 안내
- 화면에는 가상 목록으로 제한된 행만 연결
- 항목별 이벤트 리스너 대신 컨테이너 이벤트 위임

## 이력 화면

- 데스크톱의 우측 패널
- 모바일의 하단 시트
- 턴, 텍스트 방향, 점수 증가, 생성 타일 표시
- 현재 커서 이전의 적용 이력과 이후의 redo 이력 구분
- 현재 스냅샷 위치 강조
- 키보드와 터치로 항목 세부 정보 열기
- 이전 보드 미리보기의 Canvas 또는 정적 미니 그리드 재사용
- 패널 닫힘과 이력 보존의 독립 처리

특정 이력 항목 직접 이동은 v2 후속 편의 기능으로 분류함.

## 애니메이션

| 전환 | 기본 시간 | 핵심 효과 |
|---|---|---|
| 이동 | 140ms | 시작·종료 좌표 보간 |
| 병합 | 이동 후 100ms | 결과 타일 확대·복귀 |
| 생성 | 120ms | 크기와 투명도 진입 |
| 점수 | 180ms | 증가값 상승·소멸 |
| undo/redo | 140ms | 상태 전환과 커서 강조 |

- 모든 시간은 디자인 토큰으로 관리
- 모션 감소에서 0ms 적용
- 탭 비활성화 뒤 복귀 시 최종 상태로 즉시 정규화
- 한 시점에 애니메이션 루프 한 개만 허용
- 렌더러 폐기 시 예약 프레임 취소

## 입력 정책

- 방향키와 선택적인 WASD 지원
- 보드에 한정한 Pointer Events, <code>touch-action</code>, 텍스트 선택 차단
- 보드 밖 시작 입력 무시
- 임계값 미만 이동 무시와 세션 상태 완전 초기화
- 가로·세로 절댓값 중 큰 축으로 방향 판정
- 애니메이션 중 최대 2개 방향의 FIFO 버퍼와 초과 입력 폐기
- 터미널 단계 진입 시 방향 큐 비우기와 게임 이동 차단
- 버튼·선택·텍스트 입력 포커스에서 게임 키 처리 차단
- <code>lostpointercapture</code>와 <code>pointercancel</code>의 입력 상태 초기화
- 이동 불가 입력의 짧고 비차단적인 피드백
- 애니메이션 중 undo·redo·새 게임·목표 변경은 대기 방향 큐 제거와 최종 프레임 정규화 후 즉시 처리

## 디자인과 반응형

- 보드 중심의 단일 게임 셸
- 데스크톱 두 열과 모바일 한 열
- 320px 최소 너비 지원
- <code>100dvh</code>와 안전 영역 반영
- 너비와 높이를 함께 고려한 보드 크기
- 844×390 가로 화면의 겹침 방지
- 새 게임을 확인이 필요한 파괴적 동작으로 표시
- 출처 정보를 보조 계층으로 축소
- 한국어 조작 레이블의 일관된 사용

## 접근성 기능

- 시작 후 게임 영역으로 포커스 이동
- 대화상자의 제목 연결, 포커스 트랩, 닫힘 후 복귀
- undo/redo의 비활성 상태와 단축키 설명
- 테마 라디오 그룹 또는 선택 상자의 선택 상태
- 이력 패널의 <code>aria-expanded</code>와 <code>aria-controls</code>
- 점수 증가와 목표 달성의 정중한 live region
- 보드 상태의 숨김 표 또는 행별 텍스트 표현
- 아이콘 버튼의 접근 가능한 한국어 이름
- 포커스 가시성과 색 대비 기준 충족
- 키보드와 스와이프 조작 안내의 게임 영역 연결
- 이력 목록 의미론과 현재 항목의 <code>aria-current</code>
- redo 분기 삭제의 정중한 상태 알림
- <code>forced-colors</code> 환경의 타일 윤곽과 포커스 대체

## 기능 완료 기준

- 네 방향 규칙과 점수 테스트 통과
- 2048·4096·8192·16384와 사용자 지정 목표 판정 통과
- undo/redo 왕복에서 전체 상태 완전 일치
- undo 뒤 새 이동의 미래 이력 제거 확인
- 승리·패배 충돌과 대화상자 중복 부재
- DPR 1·2·3에서 Canvas 선명도 확인
- 세로·가로 모바일과 데스크톱의 겹침 부재
- 키보드 전용 흐름과 모션 감소 흐름 완료
- 10,000개 이력에서 화면 DOM 수 상한 유지
